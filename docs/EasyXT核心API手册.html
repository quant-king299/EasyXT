一快速开始

1.1 获取 API 实例

所有功能通过 get_api() 获取的单例入口访问：

from easy_xt import get_api

api = get_api()          # 获取全局单例
api2 = get_api()         # 返回同一个实例
assert api is api2       # True

1.2 初始化数据服务

获取行情数据前，需要初始化数据连接：

# 初始化数据服务（自动选择最佳数据源）
api.init_data()
# DataAPI 内部会依次尝试: QMT → TDX → 东方财富
数据源选择逻辑：init_data() 自动检测 QMT 是否可用（xtquant 是否安装且 QMT 已启动）。如果不可用，依次降级到通达信（TDX）和东方财富。初始化成功后会打印 [OK] Using XXX as data source。

1.3 最小可运行示例

from easy_xt import get_api

api = get_api()
api.init_data()

# 获取平安银行最近 10 根日线
df = api.get_price(['000001.SZ'], period='1d', count=10)
print(df)

二行情数据

get_price(codes, period, count, start, end, adjust, fill_data)

获取股票历史 K 线数据，支持多周期和多复权类型。自动走多源降级链（QMT → TDX → 东方财富）。

参数类型默认值说明
codes str | List[str] 必填 股票代码，支持单只或列表
period str '1d' K 线周期：'1m'/'5m'/'15m'/'30m'/'1h'/'1d'
count int None 获取最近 N 根 K 线（与 start/end 二选一）
start str None 起始日期 'YYYYMMDD' 或 'YYYY-MM-DD'
end str None 结束日期，不填默认今天
adjust str 'none' 复权类型：'none'/'front'/'back'/'front_ratio'/'back_ratio'
fill_data bool True 是否填充停牌日期的缺失数据

返回值：pd.DataFrame，包含 open/high/low/close/volume/amount 等标准 OHLCV 列。

示例：

# 获取最近 20 根日线（前复权）
df = api.get_price(['000001.SZ', '600519.SH'], period='1d', count=20, adjust='front')

# 获取指定日期范围的 5 分钟线
df = api.get_price(['000001.SZ'], period='5m',
                   start='20260101', end='20260115')

# 不复权原始数据
df = api.get_price(['000001.SZ'], period='1d', count=100, adjust='none')
复权类型说明：
参数值说明推荐场景
'none'不复权，原始交易价格查看真实成交价
'front'前复权，以最新价为基准调整历史看 K 线图
'back'后复权，以上市首日为基准调整当前计算真实收益率
'front_ratio'等比前复权，乘法因子量化回测（推荐）
'back_ratio'等比后复权，乘法因子复合收益曲线

2.2 get_current_price — 实时快照

get_current_price(codes)

获取股票实时行情快照（最新价、涨跌幅、成交量等）。

参数类型默认值说明
codes str | List[str] 必填 股票代码

返回值：pd.DataFrame，含 code/price/high/low/volume/pct_change 等列。

# 获取多只股票实时价格
df = api.get_current_price(['000001.SZ', '600519.SH', '300750.SZ'])
for _, row in df.iterrows():
    print(f"{row['code']}: {row['price']:.2f} ({row['pct_change']:+.2f}%)")

2.3 get_order_book — 盘口数据

get_order_book(codes)

获取买卖五档盘口数据。需 QMT 在线。

book = api.get_order_book(['000001.SZ'])
# 返回 Dict，含 bid1-bid5（买盘）和 ask1-ask5（卖盘）的价格与数量

2.4 get_l2_quote — Level-2 行情

get_l2_quote(codes)

获取 Level-2 十档行情数据。需 QMT 在线且订阅 L2 行情。

l2 = api.get_l2_quote(['000001.SZ'])
# 返回 Dict[str, Dict]，含 10 档买卖盘口

三财务数据

get_financial_data(codes, tables, start, end, report_type)

获取上市公司财务报表数据。底层调用 QMT xtdata.get_financial_data()，需 QMT 在线。

参数类型默认值说明
codes str | List[str] 必填 股票代码
tables List[str] ['Balance', 'Income', 'CashFlow'] 报表类型列表
start str '20200101' 起始报告期
end str today 结束报告期
report_type str 'report_time' 'report_time'（报告期）或 'announce_time'（公告期）

返回值：Dict[str, Dict[str, pd.DataFrame]]，结构为 {股票代码: {表名: DataFrame}}。

# 获取平安银行近 5 年三大报表
data = api.get_financial_data(['000001.SZ'],
    tables=['Balance', 'Income', 'CashFlow'],
    start='20200101')

# 取利润表
income = data['000001.SZ']['Income']
print(income[['revenue', 'net_profit_incl_min_int_inc']].head())
注意：QMT 的 get_financial_data() 只返回 Balance/Income/CashFlow/Capitalization 四种表，不包含分红明细。如需每股分红数据，使用 EasyXT 的红利低波策略内置的 get_dividend_data() 函数（自动 DuckDB → QMT get_divid_factors 降级）。

3.2 财务报表字段说明

表名中文关键字段
Income利润表revenue（营收）、net_profit_incl_min_int_inc（归母净利润）、oper_profit（营业利润）、s_fa_eps_basic（基本每股收益）
Balance资产负债表tot_assets（总资产）、tot_liab（总负债）、total_equity（股东权益）
CashFlow现金流量表net_cash_flows_oper_act（经营活动现金流）、net_cash_flows_inv_act（投资活动现金流）
Capitalization股本结构总股本、流通股本等

四股票列表与交易日

get_stock_list(sector)

获取指定板块的股票代码列表。

参数类型默认值说明
sector str None 板块名称：'沪深300'/'中证500'/'创业板'/'科创板' 等。为 None 时返回全部 A 股

返回值：List[str] 股票代码列表。

4.2 get_trading_dates

get_trading_dates(start, end)

# 获取 2025 年全年交易日
dates = api.get_trading_dates('20250101', '20251231')

五交易接口

5.1 init_trade — 连接交易服务

init_trade(qmt_path, session_id)

初始化 QMT 交易连接。需 QMT 客户端已启动并登录。

参数类型默认值说明
qmt_path str 必填 QMT userdata_mini 目录路径
session_id str 必填 会话标识，用于区分多个策略实例（建议每个策略用唯一 ID）

返回值：bool，连接成功返回 True。

api.init_trade(r'D:/QMT交易端/userdata_mini', session_id=99)
# 日志: [OK] 交易服务连接成功

5.2 add_account — 添加账户

add_account(account_id, account_type)

添加交易资金账户。

参数类型默认值说明
account_id str 必填 资金账号（在 QMT 客户端中查看）
account_type str 'STOCK' 账户类型：'STOCK'（股票）或 'CREDIT'（信用/两融） 
api.add_account('你的资金账号', 'STOCK')

5.3 buy — 买入

buy(account_id, code, volume, price, price_type)

提交买入委托。

参数类型默认值说明
account_idstr必填资金账号
codestr必填股票代码（如 '000001.SZ'）
volumeint必填买入数量（股），A 股为 100 的整数倍
pricefloat必填委托价格，传 0 为市价
price_typestr'limit''limit'（限价）或 'market'（市价）

返回值：成功返回委托编号（str），失败返回 None。

# 限价买入 100 股平安银行 @ 12.50
order_id = api.trade.buy('你的资金账号', '000001.SZ',
                         volume=100, price=12.50, price_type='limit')

5.4 sell — 卖出

sell(account_id, code, volume, price, price_type)

提交卖出委托。参数同 buy()。

# 市价卖出 100 股平安银行
order_id = api.trade.sell('你的资金账号', '000001.SZ',
                          volume=100, price=0, price_type='market')
卖出数量注意：实际可卖数量以 get_positions() 返回的 can_use_volume（可用数量）为准，而非 volume（总持仓）。部分持仓可能已被冻结（挂单中），用 volume 卖出可能废单。

5.5 cancel_order — 撤单

cancel_order(account_id, order_id)

撤销指定委托单。

api.trade.cancel_order('你的资金账号', order_id)

六账户与持仓

get_account_asset(account_id)

获取账户资产信息。

返回值：Dict，包含以下字段：

字段类型说明
account_idstr资金账号
cashfloat可用资金
frozen_cashfloat冻结资金（挂单占用）
market_valuefloat持仓总市值
total_assetfloat总资产（现金 + 市值）
asset = api.trade.get_account_asset('你的资金账号')
print(f"可用资金: {asset['cash']:,.2f}")
print(f"持仓市值: {asset['market_value']:,.2f}")
print(f"总资产:   {asset['total_asset']:,.2f}")

6.2 get_positions

get_positions(account_id, code)

获取持仓信息。

参数类型默认值说明
account_idstr必填资金账号
codestr | NoneNone股票代码，为 None 时返回全部持仓

返回值：pd.DataFrame，包含以下列：

列名说明
code股票代码
volume总持仓数量
can_use_volume可用数量（可卖）
open_price开仓均价
market_value持仓市值
frozen_volume冻结数量（挂单中）
# 获取全部持仓
positions = api.trade.get_positions('你的资金账号')
print(positions[['code', 'can_use_volume', 'market_value']])

# 获取单只股票持仓
pos = api.trade.get_positions('你的资金账号', '000001.SZ')

七数据源架构

7.1 多源降级链

EasyXT 的数据获取采用自动降级策略，无需用户手动切换数据源：

QMT (xtquant)
→
通达信 (TDX)
→
东方财富
→
兜底备份
数据源优先级覆盖范围需要条件
QMT (xtquant) 1（首选） 全部 A 股、指数、ETF、可转债 QMT 客户端已启动，xtquant 已安装
通达信 (TDX) 2（降级） 股票、指数 pip install pytdx，无需 QMT
东方财富 3（降级） 股票、ETF pip install requests
兜底备份 4（最后） 本地缓存数据 DuckDB 中有预下载数据
降级日志示例： 
[Connecting to data source...]
  Trying QMT (xtquant)...
  QMT not available, trying TDX...
  [OK] Using TDX as data source

用户无需任何干预，系统自动选择最佳可用数据源。

7.2 get_price_with_fallback

get_price_with_fallback(codes, period, count, start, end, adjust)

与 get_price() 参数相同，但显式走降级链并返回来源信息。适合需要知道"数据从哪来"的场景。

返回值：Tuple[pd.DataFrame, str]，第二个元素为数据源名称（'QMT'/'TDX'/'EASTMONEY'/'BACKUP'）。

df, source = api.get_price_with_fallback(['000001.SZ'], count=10)
print(f"数据来源: {source}")  # QMT / TDX / EASTMONEY / BACKUP

7.3 数据源状态

get_fallback_stats()

获取各数据源的调用统计，用于监控和调试。

stats = api.get_fallback_stats()
# 返回: {'QMT': {'hits': 150, 'errors': 2}, 'TDX': {...}, ...}

八完整示例

以下是一个完整的端到端示例：从获取行情到提交订单。

from easy_xt import get_api

# ========== 1. 初始化 ==========
api = get_api()
api.init_data()

# ========== 2. 获取行情 ==========
df = api.get_price(['000001.SZ'], period='1d', count=60, adjust='front_ratio')
current_price = df['close'].iloc[-1]
ma20 = df['close'].rolling(20).mean().iloc[-1]

print(f"000001.SZ 最新价: {current_price:.2f}")
print(f"20日均线: {ma20:.2f}")

# ========== 3. 简单信号 ==========
signal = 'buy' if current_price > ma20 else 'sell'

# ========== 4. 连接交易（实盘需要） ==========
if signal == 'buy':
    api.init_trade(r'D:/QMT交易端/userdata_mini', session_id=99)
    api.add_account('你的资金账号', 'STOCK')

    # 检查资金
    asset = api.trade.get_account_asset('你的资金账号')
    print(f"可用资金: {asset['cash']:,.0f}")

    # 下单
    if asset['cash'] > current_price * 100 * 1.01:
        order_id = api.trade.buy('你的资金账号', '000001.SZ',
                                 volume=100, price=current_price,
                                 price_type='limit')
        if order_id:
            print(f"买入委托成功: {order_id}")
        else:
            print("买入失败")
    else:
        print("资金不足")
else:
    print("卖出信号，暂不操作")
关键要点回顾：
  • 调用交易接口前必须先 init_trade() + add_account()
  • 买入前检查可用资金，卖出前检查可用持仓
  • session_id 每个策略实例应唯一，避免冲突
  • 实盘前先用模拟盘或 dry_run 模式验证信号逻辑
  • 价格取整使用 round_price()：股票精确到 0.01，ETF/可转债精确到 0.001
