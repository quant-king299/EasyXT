## 一、当前作用与定位

### 1.1 核心职责

`TdxDataProvider` 是 **EasyXT 实时数据架构的备用数据源**，扮演以下角色：

```
┌─────────────────────────────────────────────────────────┐
│              EasyXT 实时数据架构                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  主要数据源（QMT）                                       │
│  └─ easy_xt.data_api.DataAPI                            │
│     └─ 用于实盘交易，直接连接 QMT 终端                   │
│                                                          │
│  备用数据源（通达信）                                     │
│  └─ TdxDataProvider ← 我们分析的这个                     │
│     └─ 用于模拟盘、测试、回测                            │
│     └─ 不依赖 QMT 终端，可独立运行                       │
│                                                          │
│  辅助数据源                                              │
│  └─ EastmoneyDataProvider, ThsDataProvider              │
│     └─ HTTP 接口，作为补充                              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 1.2 使用场景

| 场景 | 使用的 Provider | 原因 |
|------|----------------|------|
| **实盘交易** | QMT DataAPI | 稳定、实时、直接下单 |
| **模拟盘/测试** | TdxDataProvider | 不依赖QMT，可测试 |
| **回测** | TdxDataProvider | 历史数据丰富 |
| **无QMT环境** | TdxDataProvider | 独立运行 |
| **策略开发** | TdxDataProvider | 快速验证 |

### 1.3 项目中的实际使用

**主要使用位置：**
1. `strategies/xueqiu_follow/core/strategy_engine.py` (第1067-1079行)
   - 雪球跟单策略获取实时价格
   - 作为 TDX 的 fallback 备用方案

2. `easy_xt/realtime_data/unified_api.py`
   - 统一数据接口的一部分
   - 作为 tdx 数据源的底层实现

3. `easy_xt/triple_source_manager.py`
   - 三数据源管理器
   - TDX 作为备用源

### 2.3 当前存在的问题

#### 问题 1：缺少 Level-2 深度数据

```python
# 当前只有 bid1/ask1
'bid1': float(quote.get('bid1', 0)) / price_divisor,
'ask1': float(quote.get('ask1', 0)) / price_divisor,
'bid1_vol': int(quote.get('bid1_vol', 0)),
'ask1_vol': int(quote.get('ask1_vol', 0)),
```

**缺失的数据：**
- bid2~bid5（买二到买五）
- ask2~ask5（卖二到卖五）
- bid_vol2~bid_vol5（买二到买五量）
- ask_vol2~ask_vol5（卖二到卖五量）

#### 问题 2：分时数据未处理基金价格

```python
# get_minute_data() 没有对基金价格除以10
'price': float(item.get('price', 0)),  # ❌ 基金价格错误！
```

#### 问题 3：K线数据未处理基金价格

```python
# get_kline_data() 也没有处理
'open': float(item.get('open', 0)),    # ❌ 基金价格错误！
'high': float(item.get('high', 0)),    # ❌ 基金价格错误！
```

#### 问题 4：缺少数据缓存机制

每次请求都要连接服务器，没有本地缓存。

#### 问题 5：连接测试不够智能

```python
# 当前测试方式
count = self.api.get_security_count(0)  # 简单但不够准确
```

#### 问题 6：缺少性能监控

- 没有请求耗时统计
- 没有服务器响应时间记录
- 没有成功率统计
