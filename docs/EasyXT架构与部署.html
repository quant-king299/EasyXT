2.2 DuckDB 存储引擎

表名内容数据来源
stock_dailyA 股日线 OHLCV（不复权）QMT 下载 / Tushare
dividend_data每股分红历史明细Tushare
cb_basic / cb_daily可转债基本信息/日行情Tushare
etf_basic / etf_dailyETF 基本信息/日行情Tushare
stock_market_cap市值数据Tushare
为什么只存不复权？复权数据在每次除权除息后全量变化，预存维护成本高。DuckDB 存原始价格，需要复权时从 QMT API 实时获取，保证准确性。

2.3 复权数据流

查询数据来源说明
不复权DuckDB stock_daily直接读取，无需 QMT
前/后复权QMT get_market_data_ex(dividend_type=...)QMT 实时计算
等比复权QMT dividend_type='front_ratio'/'back_ratio'推荐回测使用

三交易链路

3.1 QMT 直连模式

miniQMT（XtMiniQmt.exe）启动并登录后，EasyXT 通过 xttrader 直连交易服务：

api.init_trade(QMT_DATA_DIR, session_id=99)  # 连接 xttrader
api.add_account(ACCOUNT_ID, 'STOCK')          # 添加账户
api.trade.buy(...)                             # 下单 → xttrader → QMT 券商柜台

3.2 大 QMT 信号桥接

当只有大 QMT（XtItClient.exe）在运行时，EasyXT 自动切换到信号桥接模式：

Python 策略
→
信号文件 (JSON)
→
QMT_SIGNAL_DIR
→
大 QMT 接收器
→
下单执行

信号文件写入 QMT_SIGNAL_DIR（.env 配置），大 QMT 的接收器每 3 秒扫描一次。

3.3 订单生命周期

阶段状态说明
1. 生成策略调用 buy()/sell()参数校验、价格取整
2. 提交发送至 QMTxttrader 或信号桥接
3. 待报券商柜台处理中等待交易所确认
4. 部成/已成部分或全部成交get_trades() 查询成交明细
5. 废单/已撤委托失败或主动撤单检查废单原因重新下单

四QMT 自动登录

EasyXT 内置自动登录模块（core/auto_login/qmt_login.py），基于 pywinauto 实现。

配置

# .env
QMT_EXE_PATH=D:/QMT交易端/bin.x64/XtMiniQmt.exe
QMT_PASSWORD=你的登录密码
QMT_EXE_PATH_BIG=D:/QMT交易端/bin.x64/XtItClient.exe   # 可选，大QMT检测用

登录流程

  1. 检测大 QMT：tasklist 快速检查 XtItClient.exe 是否运行（<0.1 秒）
  2. 大 QMT 已运行：跳过登录，直接使用（miniQMT 与大 QMT 共享后端）
  3. 大 QMT 未运行：启动 miniQMT → 等待窗口 → 填写密码 → 回车登录
安全提醒：密码明文存储在 .env 中。建议不要在共享电脑上使用，定期修改密码。

五跨平台支持

5.1 xqshare 远程连接

Mac/Linux 用户通过 xqshare 远程连接 Windows 上的 QMT：

Mac / Linux
EasyXT
→
xqshare 客户端
↓ 网络 ↓
Windows（需 QMT + xqshare 服务端）
xqshare 服务端
→
QMT xtdata/xttrader

配置

# Mac/Linux 的 .env
XQSHARE_REMOTE_HOST=192.168.1.100    # Windows 机器 IP
XQSHARE_REMOTE_PORT=18812            # 默认端口无需修改

5.2 Mac/Linux 部署

步骤操作
1Windows 端：pip install xqshare 并启动服务端
2Windows 端：确保 QMT 已启动并登录
3Mac/Linux 端：pip install easyxt
4Mac/Linux 端：配置 XQSHARE_REMOTE_HOST
5正常使用 EasyXT API，数据自动走 xqshare 代理
性能注意：xqshare 通过 TCP 连接，延迟取决于网络质量。局域网内延迟 <10ms，远程 VPN 约 50-200ms。对于日频策略完全够用。

6.2 进程守护

策略进程需要 24×7 运行，建议用以下方案守护：

方案适用命令
NSSMWindows 服务（推荐）nssm install EasyXTStrategy
任务计划程序定时触发型策略设置每天 09:30 触发
systemdLinux (xqshare 模式)systemctl enable easyxt-strategy
supervisorLinux/Mac 通用配置 autorestart=true
NSSM 配置示例： 
nssm install EasyXTStrategy
# Application: C:\Python311\python.exe
# Arguments: D:\EasyXT\strategies\scheduler\strategy_scheduler.py
# Working directory: D:\EasyXT
# 在"Exit actions"标签设置 Restart=Yes, Delay=30000

6.3 监控与告警

生产环境建议监控以下指标：

监控项方法告警条件
进程存活NSSM / supervisor 自动重启进程退出后 30 秒未恢复
QMT 连接定时 ping xtdata 数据连续 3 次获取失败
日志异常监控 ERROR 级别日志5 分钟内 >10 条 ERROR
账户异常每日检查 get_account_asset()总资产日变化 >10%
磁盘空间系统监控磁盘使用 >80%
交易日前检查清单：
  1. QMT 客户端是否正常运行并已登录
  2. DuckDB 数据是否已更新到最新交易日
  3. 策略进程是否正常运行
  4. 账户资金是否充足（预留足够的保证金）
  5. 日志中是否有未处理的错误
