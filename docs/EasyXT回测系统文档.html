二数据准备

2.1 DuckDB 数据下载

回测系统依赖 DuckDB 中的历史行情数据。数据下载方式：

  1. GUI 下载（推荐）：python run_gui.py → "数据管理" → "下载A股数据"
  2. Tushare 下载：GUI "Tushare下载" 标签页，选"日线行情"批量下载
  3. 大 QMT 导入：先在 QMT 客户端补充数据，再"从大QMT导入"

数据文件默认路径：D:/StockData/stock_data.ddb

数据覆盖建议：回测需要覆盖完整的回测区间。如果要回测 2020-2025 年，确保 DuckDB 中 stock_daily 表的日期范围包含 2019-01-01（需要至少 1 年预热数据用于计算均线等技术指标）。

2.2 DataManager 初始化

from easyxt_backtest import DataManager

三编写策略

3.1 策略函数接口

回测引擎要求策略是一个函数，签名为：

strategy_func(trade_date, data_manager, current_positions) → dict

参数类型说明
trade_datestr当前调仓日期，格式 'YYYYMMDD'
data_managerDataManager数据管理器，提供 get_price() 等方法
current_positionsDict[str, int]当前持仓 {code: shares}

返回值：Dict[str, float]，目标权重 {code: weight}，权重之和应为 1.0。

返回空字典 {} 表示清仓（全部卖出）。

3.2 MA 双均线策略示例

经典的 MA5 上穿 MA20 买入，持有至死叉卖出：

def ma_cross_strategy(trade_date, data_manager, current_positions):
    """MA5 上穿 MA20 买入，下穿卖出"""
    import pandas as pd

    # 从股票池中选择
    stock_pool = ['000001.SZ', '600519.SH', '000858.SZ', '601318.SH', '000333.SZ']

    buy_list = []
    for code in stock_pool:
        # 获取最近 60 天数据（足够算均线）
        df = data_manager.get_price([code],
            start=(pd.Timestamp(trade_date) - pd.Timedelta(days=120)).strftime('%Y%m%d'),
            end=trade_date, period='1d', adjust='front_ratio')

        if len(df) < 20:
            continue

        close = df['close']
        ma5 = close.rolling(5).mean().iloc[-1]
        ma20 = close.rolling(20).mean().iloc[-1]
        prev_ma5 = close.rolling(5).mean().iloc[-2]
        prev_ma20 = close.rolling(20).mean().iloc[-2]

        # 金叉买入
        if prev_ma5 <= prev_ma20 and ma5 > ma20:
            buy_list.append(code)

    # 等权分配
    if buy_list:
        weight = 1.0 / len(buy_list)
        return {code: weight for code in buy_list}

    return {}  # 无信号，清仓

四运行回测

4.1 BacktestEngine 参数

BacktestEngine(initial_cash, data_manager, commission, slippage, benchmark)

参数类型默认值说明
initial_cashfloat必填初始资金（元），如 1000000
data_managerDataManager必填数据管理器实例
commissionfloat0.00025手续费率，默认万 2.5
slippagefloat0.001滑点，默认 0.1%
benchmarkstr'000300.SH'基准指数代码

4.2 回测配置详解

from easyxt_backtest import DataManager, BacktestEngine

# 1. 准备数据
dm = DataManager()

# 2. 创建引擎
engine = BacktestEngine(
    initial_cash=1000000,      # 100 万初始资金
    data_manager=dm,
    commission=0.00025,        # 万 2.5 手续费
    slippage=0.001,            # 0.1% 滑点
    benchmark='000300.SH'      # 沪深 300 基准
)

# 3. 运行回测
result = engine.run_backtest(
    strategy=ma_cross_strategy,
    start_date='2024-01-01',
    end_date='2024-12-31',
    rebalance_freq='monthly',  # 月度调仓
    stock_pool=['000001.SZ', '600519.SH', '000858.SZ', '601318.SH', '000333.SZ']
)

# 4. 查看结果
result.print_summary()
result.plot_equity_curve()

4.3 调仓与再平衡

参数值说明适用场景
'daily'每日调仓高频策略、日内信号
'weekly'每周调仓中频轮动
'monthly'每月调仓（推荐）大多数因子策略
'quarterly'每季调仓低频价值策略
调仓频率与换手率的关系：更高频的调仓会增加换手率和交易成本。月度调仓是 A 股因子策略的最佳平衡点——既能及时捕捉因子 alpha，又不会产生过高交易成本。

五绩效分析

5.1 核心指标

回测完成后，result 对象提供以下关键绩效指标：

指标英文计算公式说明
累计收益率Total Return(最终净值 - 1) × 100%回测期间的总收益
年化收益率CAGR(最终净值)^(252/交易日数) - 1年化复合增长率
年化波动率Volatility日收益标准差 × sqrt(252)风险度量
夏普比率Sharpe Ratio(年化收益 - 无风险利率) / 年化波动率风险调整后收益，>1 为良好
最大回撤Max Drawdownmax(峰值 - 谷值) / 峰值历史最大亏损幅度
胜率Win Rate盈利调仓次数 / 总调仓次数每次调仓的盈利概率
盈亏比Profit/Loss Ratio平均盈利 / 平均亏损大于 1 说明盈亏结构良好
换手率Turnover每次调仓的买卖金额占比过高则交易成本侵蚀收益
信息比率IR超额收益 / 跟踪误差相对基准的超额稳定性

6.3 因子中性化

在构建多因子模型时，某些因子可能被行业或市值风格所主导。例如"低市盈率"因子可能天然偏向银行行业。中性化处理可以剥离这些干扰：

中性化方式方法说明
行业中性化因子值对行业哑变量回归，取残差消除行业偏好
市值中性化因子值对 log(市值) 回归，取残差消除大小盘偏好
行业+市值双中性化同时对两者回归取残差最常用的配置

# 信号生成 → 勾选 "行业中性化" + "市值中性化" → 生成信号

七进阶主题

7.1 多因子模型

将多个因子组合成一个综合信号：

组合方式方法优点缺点
等权组合所有因子权重相同简单、稳健好因子权重不足
IC 加权按历史 IC 值分配权重给有效因子更高权重可能过拟合
ICIR 加权按 IC 信息比率分配权重同时考虑有效性和稳定性需要足够长的 IC 序列
排名融合对每个因子的排名加权求和量纲无关、极端值鲁棒丢失原始因子值信息

7.2 ML 选股集成

easyxt_backtest.ml 模块提供了机器学习辅助选股能力：

from easyxt_backtest.ml import MLStrategy

# 使用随机森林分类器预测下期涨跌
ml = MLStrategy(
    model='random_forest',      # 模型：random_forest / xgboost / lgbm
    factors=['alpha001', 'alpha002', 'alpha006', 'alpha012'],
    lookback=60,                # 用过去 60 天数据训练
    retrain_freq='monthly'      # 每月重新训练
)

# 在回测中使用 ML 策略
result = engine.run_backtest(strategy=ml.predict, ...)

7.3 行业/市值中性化

为什么需要中性化？以市盈率因子为例：

场景中性化前中性化后
选股结果90% 银行股 + 10% 其他各行业均衡分布
行业集中度极高（银行一家独大）合理分散
回撤风险银行板块下跌则策略崩盘行业分散降低风险
因子纯度PE 因子夹杂了行业偏好PE 因子更纯
建议：大多数多因子模型应使用行业+市值双中性化。只有当你确信你的因子与行业/市值无关（如纯粹的量价因子），才无需中性化。
