一策略开发基础

1.1 策略文件结构

一个标准的 EasyXT 策略由两个文件组成：

文件职责示例
策略核心策略类，包含信号生成、风控逻辑dividend_lowvol.py
运行入口参数配置、API 初始化、环境读取run_dividend_lowvol.py
分离的好处：核心策略可以独立测试和回测，运行入口专注于参数和部署。同一个策略核心可以被命令行、GUI、调度器三种方式调用。

1.2 最小策略模板

核心策略类的最小实现：

import pandas as pd
import numpy as np
from typing import Tuple, Dict, List

class MyStrategy:
    """我的策略"""

    def __init__(self, api=None, account_id=None, **kwargs):
        self.api = api
        self.account_id = account_id
        self.buy_n = kwargs.get('buy_n', 10)
        self.positions = None

    def generate_signals(self) -> Tuple[pd.DataFrame, pd.DataFrame]:
        """生成买卖信号。
        Returns:
            buy_list: DataFrame with code, price, ...
            sell_list: DataFrame with code, price, reason ...
        """
        # === 1. 获取数据 ===
        # === 2. 计算信号 ===
        # === 3. 返回买卖列表 ===
        return pd.DataFrame(), pd.DataFrame()

    def run(self, dry_run: bool = True):
        """运行策略"""
        buy_list, sell_list = self.generate_signals()

        # 打印信号
        print(f"买入: {len(buy_list)} 只")
        print(f"卖出: {len(sell_list)} 只")

        if not dry_run:
            self._execute_orders(buy_list, sell_list)

    def _get_positions(self) -> pd.DataFrame:
        """获取当前持仓"""
        if not self.account_id or not self.api:
            return pd.DataFrame()
        return self.api.trade.get_positions(self.account_id)

    def _execute_orders(self, buy_list, sell_list):
        """执行买卖"""
        for _, row in sell_list.iterrows():
            self.api.trade.sell(self.account_id, row['code'],
                volume=100, price=row['price'], price_type='limit')
        for _, row in buy_list.iterrows():
            self.api.trade.buy(self.account_id, row['code'],
                volume=100, price=row['price'], price_type='limit')

二数据获取

2.1 DuckDB 优先策略

策略中的数据获取应采用"DuckDB 优先、QMT 降级"的模式：

def get_stock_prices() -> Dict[str, float]:
    """获取全市场最新收盘价，优先 DuckDB，降级 QMT"""
    # 方式1：DuckDB（毫秒级）
    try:
        import duckdb
        con = duckdb.connect('D:/StockData/stock_data.ddb', read_only=True)
        latest = con.execute("SELECT MAX(date) FROM stock_daily").fetchone()[0]
        rows = con.execute(f"""
            SELECT stock_code, close FROM stock_daily
            WHERE date = '{latest}' AND close > 0
        """).fetchall()
        con.close()
        if rows:
            return {r[0]: r[1] for r in rows}
    except Exception:
        pass

    # 方式2：QMT API 降级（秒级）
    try:
        from easy_xt import get_api
        api = get_api()
        api.init_data()
        df = api.get_current_price(api.get_stock_list())
        return dict(zip(df['code'], df['price']))
    except Exception:
        return {}

2.2 QMT 降级模式

EasyXT 框架自动处理数据源降级。在策略中直接使用 api.get_price()，框架会自动选择最佳数据源。

性能对比：DuckDB 全市场收盘价查询 ~50ms，QMT API 同样查询 ~3-5 秒。高频策略应尽量用 DuckDB 预下载数据。

2.3 分红/财务数据

分红数据通过 DuckDB dividend_data 表获取，无数据时降级到 QMT get_divid_factors()：

def get_dividend_data() -> pd.DataFrame:
    try:  # DuckDB
        import duckdb
        con = duckdb.connect('D:/StockData/stock_data.ddb', read_only=True)
        df = con.execute("""SELECT ts_code as code, cash_div, ex_date
            FROM dividend_data WHERE cash_div > 0 ORDER BY ex_date DESC""").fetchdf()
        con.close()
        if not df.empty: return df
    except: pass

    # 降级 QMT
    from xtquant import xtdata
    results = []
    for code in stock_list:
        data = xtdata.get_divid_factors(code, '20150101', '20991231')
        if data is not None and not data.empty:
            last = data.sort_values('time').iloc[-1]
            if float(last['interest']) > 0:
                results.append({'code': code, 'cash_div': float(last['interest'])})
    return pd.DataFrame(results)

三信号生成

3.1 单因子信号

最简单的信号生成——按某个指标排名选股：

# 按股息率降序选前 N 只
pool['yield_rank'] = pool['dividend_yield'].rank(ascending=False)
buy_list = pool.nsmallest(buy_n, 'yield_rank')

3.2 多因子排名融合

对每个因子分别排名，加权求和得到综合得分。推荐使用排名法而非原始值法——排名法量纲无关、极端值鲁棒：

# 因子1：股息率（越高越好 → 降序排名）
pool['yield_rank'] = pool['dividend_yield'].rank(ascending=False)

# 因子2：波动率（越低越好 → 升序排名）
pool['vol_rank'] = pool['volatility'].rank(ascending=True)

# 因子3：市值（越小越好 → 升序排名）
pool['size_rank'] = pool['market_cap'].rank(ascending=True)

# 综合得分（权重可调）
pool['score'] = (pool['yield_rank'] * 1.0 +
                 pool['vol_rank'] * 0.5 +
                 pool['size_rank'] * 0.3)

# 得分最低的 N 只入选
selected = pool.nsmallest(buy_n, 'score')
为什么不直接加权原始值？股息率 2%-15%，波动率 15%-50%，直接用原始值加权会把波动率的影响放大 10 倍。排名打分是量贩策略的通用做法。

3.3 技术指标信号

结合 MA、RSI、MACD 等技术指标生成交易信号：

def generate_ma_signal(code, price_map):
    closes = get_historical_closes(code, period=60)
    if len(closes) < 20: return None

    ma5 = np.mean(closes[-5:])
    ma20 = np.mean(closes[-20:])
    prev_ma5 = np.mean(closes[-6:-1])
    prev_ma20 = np.mean(closes[-21:-1])

    # 金叉买入
    if prev_ma5 <= prev_ma20 and ma5 > ma20:
        return 'buy'
    # 死叉卖出
    elif prev_ma5 >= prev_ma20 and ma5 < ma20:
        return 'sell'
    return None

4.2 价格取整

不同品种有不同的最小价格变动单位，必须取整否则废单：

def round_price(code, price):
    """按交易品种取整到交易所最小价差"""
    code_str = str(code).replace('.SH','').replace('.SZ','')
    if code_str.startswith(('1','12')):     # 可转债 → 0.001
        return round(price, 3)
    elif code_str.startswith(('5','15','16','588')):  # ETF → 0.001
        return round(price, 3)
    else:                                    # 股票 → 0.01
        return round(price, 2)

4.3 卖出数量安全处理

卖出时取 enable_amount（可用数量）而非 volume（总持仓），避免挂单中的持仓被重复卖出导致废单：

def get_safe_sell_volume(positions, code, default=100):
    """获取安全的卖出数量"""
    match = positions[positions['code'].str.contains(code.replace('.SH','').replace('.SZ',''))]
    if match.empty: return default
    row = match.iloc[0]
    enable = int(row.get('enable_amount', 0) or row.get('can_use_volume', 0) or 0)
    if enable > 0: return enable
    volume = int(row.get('volume', 0) or 0)
    return max(volume, default) if volume > 0 else default

五实盘部署

5.1 dry_run 安全模式

所有策略必须支持 dry_run 模式——只输出信号，不实际下单：

# 安全模式（不下单）
python strategies/quant_strategies/run_dividend_lowvol.py

# 实盘模式
python strategies/quant_strategies/run_dividend_lowvol.py --trade
铁律：实盘前必须先用 dry_run 模式跑至少 1-2 天，确认信号逻辑、资金计算、买卖数量都正确。

5.2 .env 配置管理

策略运行入口从 .env 读取敏感配置，不硬编码：

def _env(key, default=''):
    """从 .env 读取配置"""
    p = os.path.join(PROJECT_ROOT, '.env')
    try:
        with open(p, encoding='utf-8') as f:
            for line in f:
                if line.startswith(f'{key}='):
                    return line.split('=', 1)[1].strip()
    except: pass
    return default

六多策略调度

6.1 StrategyCoordinator

多策略协调器负责：统一初始化交易连接、按比例分配资金、去重合并订单、先卖后买执行。

调度流程
StrategyCoordinator
→
逐个策略 generate_signals()
→
订单去重
→
资金分配
→
先卖后买

关键能力：

  • 订单去重：同一只证券只能被一个策略交易，避免买卖冲突
  • 资金分配：按策略的 target_allocation 比例分配可用资金
  • 虚拟持仓：各策略独立追踪持仓，不与 QMT 真实持仓混淆

6.2 VirtualBookkeeper

虚拟账簿记录每个策略的独立持仓，策略调用 get_positions() 时返回虚拟持仓而非 QMT 真实持仓：

# 协调器注入虚拟持仓
strategy.positions = virtual_bookkeeper.get_positions(strategy_name)

# 策略内 get_positions() 优先用虚拟持仓
def _get_positions(self):
    if self.positions is not None:      # 被注入的虚拟持仓
        return self.positions
    return self.api.trade.get_positions(self.account_id)  # 回退 QMT

七内置策略解析

剖析 EasyXT 内置策略的实现，帮助理解策略开发模式。

7.1 红利低波策略

环节实现
数据DuckDB stock_daily（股价）+ dividend_data（分红），降级 QMT get_divid_factors
股息率每股最新分红 / 当前股价 × 100%
波动率60 日对数收益率年化标准差
门槛股息率 ≥ 2%，波动率 ≤ 50%
打分股息率排名 × 1.0 + 波动率排名 × 0.5
买入得分最低的前 20 只，不重复当前持仓
卖出持仓跌出前 40 名时清仓

7.2 可转债双低/三低策略

策略公式逻辑
双低转债价格 + 转股溢价率越小越好，代表偏债（保护）和偏股（进攻）的平衡
三低双低值 + 剩余规模惩罚在双低基础上排除大规模债

均线多头排列过滤器：MA5 > MA10 > MA20 > MA30，每满足一层 +25 分，低于 50 分不买。

7.3 ETF 趋势策略

环节实现
选池主要 ETF（沪深300、创业板、科创50、恒生等）
趋势判断收盘价在 MA20 之上为上升趋势
排序按近期涨幅排名，买入最强的前 N 只
止损跌破 MA20 或回撤超过 8% 清仓
学习路径建议：先读懂红利低波（逻辑最清晰），再看 ETF 趋势（技术指标入门），最后研究可转债双低（多因子融合）。三个策略覆盖了基本面因子、技术因子、多品种三大方向。
