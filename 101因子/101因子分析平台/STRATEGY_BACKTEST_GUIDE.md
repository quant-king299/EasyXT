## 📖 使用说明

### 1. 打开平台

启动后，在左侧导航栏选择 **"🎯 策略回测"**

### 2. 配置参数

#### 回测时间
- **开始日期**: 回测起始日期（默认：2015-01-01）
- **结束日期**: 回测结束日期（默认：2024-12-31）

#### 策略参数
- **选股数量**: 每次调仓选择的股票数量（1-50只，默认5只）

## 💡 策略说明

### 小市值策略

**策略逻辑**：
1. 每月第一个交易日调仓
2. 从股票池中选择流通市值最小的N只股票
3. 等权重配置
4. 持有到下一个调仓日

**数据来源**：
- 价格数据：DuckDB（优先）→ QMT → Tushare
- 市值数据：QMT实时计算 → 缓存 → Tushare

**适用场景**：
- A股市场
- 中长期投资
- 小市值风格暴露

## ❓ 常见问题

### Q1: 回测失败怎么办？

**A**: 检查以下几点：
1. 确保easyxt_backtest在正确路径
2. 检查DuckDB数据库是否存在
3. 查看错误信息了解具体原因

### Q2: 可以修改策略吗？

**A**: 可以！您可以：
1. 在`easyxt_backtest/strategies/`中创建新策略
2. 修改`strategy_backtest_page.py`中的策略导入
3. 重启应用

### Q3: 数据从哪里来？

**A**: 数据优先级：
1. DuckDB（本地，最快）
2. QMT（迅投终端）
3. Tushare（在线API）

### Q4: 可以回测其他策略吗？

**A**: 目前支持：
- ✅ 小市值策略
- ✅ 任何继承自`StrategyBase`的策略

您可以轻松扩展其他策略！

## 📝 开发说明

### 添加新策略页面

1. 在`src/workflow/`中创建新页面：
```python
# your_strategy_page.py
def render_your_strategy_page():
    st.title("您的策略")
    # 策略逻辑
```

2. 在`main_app.py`中添加导航：
```python
elif page == "🎲 您的策略":
    from src.workflow.your_strategy_page import render_your_strategy_page
    render_your_strategy_page()
```

3. 重启应用

### 修改策略参数

编辑`src/workflow/strategy_backtest_page.py`中的`render_config_panel()`函数。
